# Post-Hackathon Changes

## Purpose

This document records changes made after the hackathon submission. The original submission state is preserved separately in the `hackathon-snapshot` branch and the `hackathon-final-submission` tag. The work documented here was intended to improve clarity, cleanup, and usability without rewriting what existed at submission time.

## Preserved Snapshot Reference

- Branch: `hackathon-snapshot`
- Tag: `hackathon-final-submission`

## Categories of Post-Hackathon Work

- README and repository presentation
- Documentation and supplementary project notes
- UI polish
- Bug fixes
- Code cleanup and structure
- Environment and setup improvements

## Specific Notable Changes

- Reorganized repository and docs structure for clarity.
- Added documentation describing engineering notes, implementation status, and post-hackathon changes.
- Updated the README to distinguish the preserved hackathon snapshot from the current `main` branch.
- Removed unused PDF.js assets, dependency/config remnants, and other dead tracked artifacts.
- Refined dashboard and role-detail UI hierarchy.
- Improved labels, breadcrumb behavior, spacing, and action placement in the dashboard flows.
- Improved card and tile contrast to make key actions and content easier to scan.
- Preserved the original hackathon submission separately while continuing cleanup work on `main`.
